## Scope of Stewardship

The founding steward is responsible for:

- spec integrity (normative documents, state machines, schemas)
- reference implementation correctness
- conformance suite maintenance
- OEP (Open Enhancement Proposal) triage and decision
- governance evolution toward a multi-stakeholder model

## Path to Broader Governance

The current single-steward posture is an honest reflection of the bootstrap phase. The intended trajectory is:

1. **Single steward (now)** — founding author, direct decision authority, all changes via atomic commits to main.
2. **Co-stewards** — external maintainers from aligned ecosystems join as the spec stabilizes. Decision via OEP review.
3. **Neutral consortium** — fiscal sponsor or lightweight legal entity holds trademarks, accepts patent pledges, and formalizes governance. Timeline tied to external review and adoption milestones.

See `governance/GOVERNANCE.md` for the full governance posture and `governance/OEP_PROCESS.md` for the enhancement proposal process.

## How to Become a Maintainer

There is no open call for additional maintainers yet. External participation currently happens through:

- **Cosigners** — bounded review of one artifact family (see `docs/COSIGNERS.md`)
- **Design partners** — fit/friction/adoption feedback (see `docs/DESIGN-PARTNERS.md`)
- **OEPs** — spec enhancement proposals (see `governance/OEP_PROCESS.md`)
- **GitHub Issues / Discussions** — bug reports, questions, discussion

Once external review produces a stable spec baseline, a formal call for co-stewards will open.
