# Roadmap

## Phase 0
- Publish spec repository
- Publish schemas and examples
- Publish MCP profile
- Publish reference scaffold

## Phase 1
- Build `@oaps/core`
- Build `@oaps/evidence`
- Build `@oaps/policy`
- Add schema-derived TypeScript types
- Add canonical serializer and hash utilities

## Phase 2
- Build `@oaps/mcp-adapter`
- Support sidecar, gateway, and embedded deployment models
- Demo policy-enforced MCP tool call with evidence output

## Phase 3
- Build `@oaps/http`
- Implement well-known discovery
- Implement interaction, approval, and evidence endpoints
- Add auth binding and idempotency handling

## Phase 4
- Add conformance tests
- Add A2A profile draft
- Add economic authorization profile work
- Add registry and governance workflows

## Phase 5
- Production-grade gateway
- Rust gateway optional, after TypeScript reference server
- Multi-profile adapters
- External contributors and co-authors
