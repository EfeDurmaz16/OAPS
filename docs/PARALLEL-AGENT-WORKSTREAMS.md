# Parallel Agent Workstreams

## Purpose

AICP is too broad to move as a single linear stream.

The intended operating model is founder-led and agent-amplified, with parallel workstreams acting like a founding team.

## Workstream Model

### 1. Spec Workstream

Responsible for:

- charter
- core foundation draft
- terminology
- normative boundaries
- profile and binding decomposition

Primary outputs:

- draft documents
- OEPs
- compatibility notes

### 2. Research Workstream

Responsible for:

- external protocol analysis
- ecosystem mapping
- standards and governance research
- market and cosigner intelligence

Primary outputs:

- comparison matrices
- protocol gap analyses
- partner and standards body research

### 3. Implementation Workstream

Responsible for:

- reference implementation evolution
- binding prototypes
- profile adapters
- schema-backed runtime alignment

Primary outputs:

- reference packages
- adapters
- proof-of-concept implementations

### 4. Conformance Workstream

Responsible for:

- fixture corpus
- TCK design
- replay and evidence test vectors
- interoperability validation

Primary outputs:

- conformance suites
- fixture packs
- test vectors

### 5. Outreach and Governance Workstream

Responsible for:

- public positioning
- cosigner targeting
- governance docs
- working group preparation

Primary outputs:

- outreach packets
- review briefs
- governance updates

## Human and Agent Responsibilities

### Founder Responsibilities

- final product and protocol judgment
- normative boundary decisions
- public positioning
- external relationship management
- final acceptance of changes

### Agent Responsibilities

- research
- first-pass document drafting
- repo analysis
- schema and fixture drafting
- prototype implementation
- partner intelligence synthesis

## Coordination Rules

- each workstream should produce reviewable artifacts, not vague notes
- workstreams should converge through OEPs and explicit decision records
- implementation should not silently outrun the written spec
- spec should not silently outrun conformance
- outreach should not get ahead of what can be defended technically

## Default Near-Term Allocation

Until a broader team exists, the default assumption is:

- you drive all final decisions
- multiple agents run in parallel across the five workstreams
- output is merged through deliberate review rather than by sequence alone
