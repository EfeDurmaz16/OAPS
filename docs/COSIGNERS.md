# AICP Cosigners

## Purpose

This page describes **how to become a cosigner** of AICP (Agent Interaction Control Protocol). No cosigners are currently signed on — this page exists to make the role and process visible so that potential cosigners can understand what is being asked before committing.

A cosigner is not a blanket endorsement for the whole suite.
It is a bounded review of one artifact family with enough confidence to keep the work moving.

## What A Cosigner Helps With

Cosigners are most useful when they can comment on:

- whether the artifact boundary is technically defensible
- whether the profile mapping preserves the adjacent ecosystem's strengths
- whether the conformance claim is honest
- whether the draft should remain draft or advance to a tighter review cycle

## Good Cosigner Targets

The best cosigner candidates are people or organizations that already care about one of these areas:

- tool and agent protocol ecosystems
- identity and attestation ecosystems
- payments and authorization ecosystems
- provisioning and lifecycle ecosystems
- commerce workflow ecosystems

## Current Cosigner Categories

| Category | Example review target family | What to ask |
| --- | --- | --- |
| MCP maintainers | MCP profile | preserve tool utility while adding governance semantics |
| A2A maintainers | A2A profile | preserve task lifecycle meaning while adding replayable evidence |
| Trust and attestation reviewers | auth-web / FIDES-TAP | keep trust boundaries explicit and upgradeable |
| Payment reviewers | x402 / MPP / AP2 | keep payment coordination rail-agnostic where intended |
| Provisioning reviewers | OSP | keep lifecycle and credential semantics defensible |
| Commerce reviewers | ACP / UCP / commerce draft | keep commerce semantics above checkout-specific detail |

## What A Cosigner Packet Should Include

- the exact artifact under review
- the decision requested
- the current implementation status
- the current conformance status
- the main open questions
- the expected follow-up path

For a reusable structure, see [`docs/REVIEW-PACKET-TEMPLATE.md`](./REVIEW-PACKET-TEMPLATE.md) and [`docs/REVIEW-READINESS-CHECKLISTS.md`](./REVIEW-READINESS-CHECKLISTS.md).

## What A Cosigner Is Not

A cosigner is not being asked to approve:

- the entire suite in one pass
- every future profile family
- implementation details that are still clearly conceptual
- unrelated governance questions
